VectorAssignment & Pong

A three-part project: a native C++ DLL for 2D/3D vector math, a console test suite that validates every export, and a Unity Pong clone that drives ball and paddle physics through PongVector — a thin C# wrapper over the DLL. The goal was to practice engine-adjacent math in C++, then prove it in a real gameplay loop without leaning on Unity’s built-in vector types for core simulation.

Project structure

  • VectorAssignment — VectorAssignment.dll exports plain C-style functions with extern "C" __declspec(dllexport) and blittable Vec2 / Vec3 structs.
  • TestOperations — Executable tests for arithmetic, normalization, dot/cross, reflection, distance, projection, and edge cases (zero vectors).
  • Pong (Unity) — Playable two-player / AI match: movement, wall reflection, paddle AABB hits, angle-based deflection, and scoring.

Native vector library

Operations are implemented in a single header-style translation unit using <math.h> (sqrtf, acosf). Each function is exported for consumption from managed code without name mangling on the C++ side.

Vec2 (used by Pong)

  • v2_add, v2_subtract, v2_scale
  • v2_magnitude, v2_normalize, v2_dot
  • v2_reflect — wall bounces use r = v − 2(v·n)n
  • v2_lerp

Vec3 (library completeness)

  • Core ops: add, subtract, scale, magnitude, normalize, dot, cross
  • v3_distance / v3_distance_sqr — squared distance avoids a sqrt when only ordering matters
  • v3_angle_between, v3_clamp_magnitude, v3_project, v3_lerp

Normalization guards divide-by-zero by returning a zero vector when magnitude is 0. Projection returns zero when the target axis has zero length.

P/Invoke bridge (PongVector.cs)

Unity talks to the DLL through a static wrapper class. The C# side mirrors the C++ layout exactly so structs pass by value without custom marshalling:

  • [StructLayout(LayoutKind.Sequential)] on Vec2 with public float x, y
  • [DllImport("VectorAssignment")] on each import — DLL placed under Assets/Plugins/ for Windows x64 builds
  • Ball and paddles store simulation state as PongVector.Vec2; only the final transform.position sync uses Unity’s Vector3 for rendering

Pong gameplay systems

BallController

  • Integrates velocity each frame with v2_add and v2_scale (delta time as scalar).
  • Top/bottom walls: build a surface normal, then v2_reflect on velocity.
  • Paddle hits: AABB overlap, snap ball outside paddle, compute hit factor along paddle height (−1 bottom → +1 top), map to a deflection angle (up to maxBounceAngle), rebuild direction, then v2_normalize + v2_scale to keep constant speed.
  • Score zones past ±9 units; GameManager increments score and resets serve direction.

PaddleController & PaddleAI

Human paddles read keyboard input into a vertical delta vector and accumulate position with v2_add. The AI paddle tracks the ball’s Y with a dead zone, moves at a capped speed, and clamps to the same court limits as the ball uses for walls.

TestOperations

A separate console project exercises the DLL before Unity depends on it. Tests cover expected numeric results and failure modes (e.g. normalizing {0,0}) so regressions in native code surface as [PASS]/[FAIL] output instead of subtle in-game bugs.

Why vectors matter in games

The Pong slice is intentionally small, but it uses the same primitives as larger engines: velocity as direction × speed, normals for bounces, dot products for angles and lighting-style queries, and (in 3D) cross products for perpendicular axes. Matrices sit one layer above — Unity’s Transform is a 4×4 matrix handling translation, rotation, scale, and eventually projection to screen space. This project keeps simulation in explicit vector ops so the math stays visible in code reviews and coursework-style documentation.

Build & run

  • DLL — Open VectorAssignment.sln, build Release x64, copy VectorAssignment.dll into Pong/Assets/Plugins/.
  • Tests — Build and run TestOperations.exe after the library project.
  • Play — W/S vs arrow keys; first to 5 points; R to restart after game over.

What this demonstrates

  • Shipping a native plugin boundary with correct struct layout and DLL lifecycle (build arch, plugin folder, import name).
  • Separating testable C++ core from presentation-layer Unity code.
  • Applying vector math to feel (reflection, angled paddle hits, speed normalization) rather than only textbook exercises.
